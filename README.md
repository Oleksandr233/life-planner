# 🎯 Планировщик жизни

Веб-приложение для планирования жизни: цели на разных горизонтах (от «сегодня» до «этого года»), привычки, привязанные к этим целям, и лёгкая геймификация, чтобы их было проще держать. Многопользовательское, с регистрацией и изоляцией данных.

**🔗 Живое демо:** https://life-planner-aj64.onrender.com

> ⚠️ Демо развёрнуто на бесплатном тарифе и «засыпает» после простоя — первое открытие может занять 30–60 секунд, пока сервис просыпается.

## Идея

Большинство трекеров — это либо календарь, либо список привычек. Здесь акцент на **связи горизонтов**: большая цель на год раскладывается на привычки, которые видно прямо под ней. Так ежедневная мелочь («растяжка 15 минут») наглядно подключена к большой цели («сесть на шпагат»).

## Возможности

- **Цели по горизонтам** — Сегодня / Эта неделя / Этот месяц / Этот год, сгруппированы на одном экране.
- **Привычки, привязанные к целям** — под каждой целью видно, какие ежедневные действия её двигают.
- **Стрики (серии)** — счётчик дней подряд с честной логикой по датам: пропустил день — серия сбрасывается.
- **Геймификация** — монеты за выполнение привычек (с бонусом за длину серии) и магазин тем оформления за накопленные монеты.
- **Аккаунты** — регистрация, вход, выход; у каждого пользователя свои цели, привычки и монеты.

## Технологии

| Область | Инструмент |
|---|---|
| Язык | Python |
| Веб-фреймворк | FastAPI |
| ORM / работа с БД | SQLModel (поверх SQLAlchemy) |
| База данных | SQLite (локально) / PostgreSQL (продакшен) |
| Аутентификация | bcrypt (хеширование паролей), Starlette SessionMiddleware (cookie-сессии) |
| Сервер | Uvicorn |
| Деплой | Render (автодеплой из GitHub) |

## Что внутри интересного технически

- **Пароли не хранятся в открытом виде** — только bcrypt-хеши; проверка идёт сверкой хеша, исходный пароль восстановить нельзя.
- **Сессии на подписанных cookie** — после входа пользователю выдаётся подписанный секретным ключом «пропуск», по которому сервер опознаёт его на каждом запросе.
- **Изоляция данных по пользователям** — все цели, привычки и монеты привязаны к `user_id`; каждое действие дополнительно проверяет, что объект принадлежит текущему пользователю (защита от доступа к чужим данным по подставленному id).
- **Одна кодовая база — две СУБД** — адрес базы читается из переменной окружения: локально это лёгкий SQLite, в продакшене — PostgreSQL, без изменений в коде.
- **Связи между таблицами (foreign keys)** — цели ↔ привычки ↔ пользователи; данные из формы не принимаются на веру (например, проверяется, что выбранная цель реально существует и принадлежит пользователю).

## Запуск локально

```bash
# 1. Клонировать репозиторий
git clone https://github.com/ВАШ_ЛОГИН/life-planner.git
cd life-planner

# 2. Создать и активировать виртуальное окружение
python -m venv venv
# Windows:
.\venv\Scripts\Activate.ps1
# macOS / Linux:
source venv/bin/activate

# 3. Установить зависимости
pip install -r requirements.txt

# 4. Запустить
fastapi dev main.py
```

После запуска приложение доступно на http://127.0.0.1:8000

Без дополнительных настроек используется локальная база SQLite — никакой установки базы данных не требуется.

## Переменные окружения

| Переменная | Назначение | По умолчанию |
|---|---|---|
| `DATABASE_URL` | Адрес базы данных. Если не задана — используется локальный SQLite. | `sqlite:///habits.db` |
| `SECRET_KEY` | Секрет для подписи пользовательских сессий. В продакшене обязательно задать своё длинное случайное значение. | заглушка для разработки |

## Планы развития

- Достижения и бейджи за серии и регулярность.
- Модуль питания с учётом калорий и модуль тренировок.
- Мобильная версия с push-напоминаниями.
- Перенос интерфейса на отдельный фронтенд (React) и разделение на API + клиент.

## Заметки

- Демо работает на бесплатном тарифе Render: сервис засыпает при простое, а бесплатная база PostgreSQL имеет ограниченный срок жизни. Для боевого использования предполагается переход на платный тариф.
- Проект учебный, разрабатывался как портфолио — от первого запроса к серверу до многопользовательского приложения в продакшене.
